## Learnings

- **Turn 7 closed the biggest standing risk**: the Playwright suite that sat built-but-unexecuted for 3 turns was finally run and passed, converting "zero bugs found across 7 tasks" from an untested claim into an actually-verified one (for the one smoke path it covers). This validates last turn's top recommendation and confirms the lesson: a built-but-unexecuted test asset should trigger `qa-engineer` immediately, not after several more `developer` tasks land on top of it.
- **Static-review-only confidence is still real but narrower than it looks**: the one passing test covers only the boards-overview happy path. Lists, cards, and cascading deletes — the bulk of what's been built (task-003, 005, 006) — are still verified by code review alone, not execution. This isn't a new problem, just a reminder the turn-7 fix was partial, not total.
- **The stale "no dependencies" architecture line has now been flagged 6 turns straight with zero action**, despite being a one-line, zero-risk edit. This reconfirms the standing hypothesis: non-blocking review notes without an assigned task number reliably rot indefinitely. It will keep getting flagged here until either a task-NNN is created for it or a future `architect`/`tech-writer` pass is asked to sweep design docs for staleness as part of its own job.
- **Core Trello-defining interactions are still entirely missing after 8 turns**: no card detail/description editing (FR8), no drag-and-drop at all (FR4/FR5/FR6/FR9 — arguably the single most identifiable "Trello" interaction), no confirm-before-delete (FR11 — every delete today is silent and irreversible). Three-level CRUD depth with full cascade delete was built before any of the interactions that make the app feel like its namesake. This ordering was a deliberate backlog choice (get something runnable early) and has paid off via clean reviews, but it means the product still can't demonstrate its own core value proposition yet.
- **`ui-tester` has never run once in 8 turns** despite a real UI existing since turn 3. The automated Playwright suite is a narrow scripted check, not a substitute for an exploratory pass that might catch layout/empty-state/interaction issues the script wasn't written to look for.
