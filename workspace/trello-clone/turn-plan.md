## Learnings

- **The last Must-have (FR8, card modal) landed clean, closing the "core interactions missing" gap for editing** — but FR11 (confirm-before-delete) has now sat TODO for 3 turns despite being cheap, dependency-satisfied, and explicitly flagged in the backlog's own Risks section as preventing real irreversible data loss. Deliberate reordering to prioritize task-007 over task-011 is defensible (both are ready, FR8 is Must vs. FR11 Should), but the risk it was meant to close (a single click silently deleting a board/list/card, no undo) is still live in the shipped app right now. This should be the top priority this turn rather than slipping further.
- **The 8-task, zero-bounce review streak is worth treating with mild skepticism, not just praise.** Every code review across 9 turns has been APPROVED first-pass. The one time dynamic verification actually ran (turn 7 Playwright), it also passed — reassuring, but it only covers ~1/8 of the built surface (boards-overview only) and has not been extended to cover task-005/006/007 (lists, cards, cascades, modal) at all. A streak this clean with this little execution-based coverage means confidence in the untested 7/8 of the app rests entirely on review quality, which itself has never been tested by a real bug.
- **`architecture.md`'s stale "no dependencies" line is now a 7-turn-old process smell**, not just a doc nit: it demonstrates that non-blocking review notes without an assigned task number reliably rot indefinitely in this project. Worth breaking the pattern by either giving it a task number or having the next `architect`/`tech-writer` touch include a doc-staleness sweep as part of its own job, rather than logging it as a footnote turn after turn.
- **`ui-tester` has never run once in 9 turns** despite the UI now including boards, lists, cards, and a modal (turn 3 through turn 8). The scripted Playwright smoke test is not a substitute for an exploratory pass — especially now that a modal with escape/overlay/save/cancel interactions exists, which is exactly the kind of surface manual exploratory testing catches issues in that a single scripted test won't.
