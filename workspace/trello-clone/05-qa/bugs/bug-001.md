# Bug 001 — List reorder, board reorder, and the FR6a click-based fallback are not wired up in the UI

## Summary

Requirements FR6 (Must — "Reorder lists within a board via drag-and-drop"), FR9
(Should — "Reorder boards on the boards overview page via drag-and-drop"), and
FR6a (Must — click-based fallback for moving/reordering cards) are not
implemented in the UI, even though the store layer already has the supporting
functions.

- `store.reorderLists(boardId, listId, toIndex)` exists
  (`04-build/src/public/js/store.js:197`) but `events.js` never wires a
  `dragstart`/`dragover`/`drop` handler for `.list-column` elements, and
  `render.js` never marks list columns `draggable="true"`.
- `store.reorderBoards(boardId, toIndex)` exists
  (`04-build/src/public/js/store.js:73`) but `.board-row` elements in
  `render.js` are not `draggable`, and no drag handlers exist for the boards
  list either.
- No "Move to..." menu/control exists anywhere in the rendered card markup
  (`render.js` `renderCards`), so there is no way to reorder/move a card
  without native drag-and-drop — FR6a explicitly requires this as a
  click-based fallback, "to be used if drag-and-drop is not yet
  implemented/working."

Only card drag-and-drop (FR4/FR5, reorder within a list and move across
lists) is implemented and works correctly (verified in
`05-qa/tests/journeys.spec.js` and the walkthrough screenshots).

## Steps to reproduce

1. Start the app (`cd 04-build && npm start`), open a board with 2+ lists.
2. Try to drag a list header/column to reorder it relative to another list —
   nothing happens (no `draggable` attribute, no drop handling).
3. From the boards overview, try to drag a board row to reorder it — same
   result.
4. Look for any button/menu on a card offering "Move to another list" or
   similar as a non-drag alternative — none exists.

## Evidence

Screenshot showing the board view with multiple lists (list columns are
plain `<div>`s, not draggable, and there is no "Move to..." affordance on any

![Lists created, no reorder affordance](../walkthrough/turn-011/04-lists-created.png)

## Impact

- FR6 is marked **Must** in requirements.md — this is a missing Must-have,
  not just a nice-to-have gap.
- FR6a is also marked **Must** and exists specifically as a safety net for
  exactly this kind of partial drag-and-drop implementation; its absence
  means there is currently no way at all to reorder lists or boards for a
  user whose only working interaction is card drag-and-drop.
- FR9 (Should) board reorder is likewise absent.

## Suggested next step (for triage, not applied by QA)

Wire `dragstart`/`dragover`/`drop` handlers for `.list-column` in
`lists-container` (mirroring the existing card drag-and-drop pattern in
`events.js`) calling `store.reorderLists`, and similarly for `.board-row` in
`boards-list` calling `store.reorderBoards`; and/or add the FR6a click
fallback control if drag-and-drop for lists/boards is intentionally being
deferred.
